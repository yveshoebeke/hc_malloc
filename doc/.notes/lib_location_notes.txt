System-Wide Library (For Sharing Across Projects)
If you want to use your library across multiple independent projects without copying the source files every time, you should install it into the standard UNIX directory structure.

	1. Recommended System Paths:
	Do not drop your files into /usr/include or /usr/lib, as these are reserved for the operating system and package managers. 
	Instead, deposit them in the /usr/local/ 
	subdirectories:
		Header Files (.h): Deposit in /usr/local/include/ - Tip: Create a subfolder like /usr/local/include/my_lib/my_lib.h to avoid name conflicts.
		Compiled Library Files (.a or .so / .dylib): Deposit in /usr/local/lib/

	2. How to Install It
		Typically, you write a Makefile that handles moving the files to these locations with root privileges:
			sudo cp libs/my_custom_lib/include/my_lib.h /usr/local/include/
			sudo cp libmy_lib.a /usr/local/lib/
	
	3. How to Compile It
		Because /usr/local/include and /usr/local/lib are automatically in GCC’s standard search paths, you can use the angle-bracket syntax #include <my_lib.h> instantly. 
		You only need to tell the linker to link the binary:
			gcc main.c -lmy_lib -o my_app
